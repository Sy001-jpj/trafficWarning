# 智慧交通监控预警系统设计系统

## 1. 产品定位

本系统定位为面向城市交通管理场景的智能监控与预警平台，核心目标是帮助管理员快速掌握交通运行状态、发现异常事件、查看设备健康情况，并通过 AI 辅助完成异常分析、拥堵识别和事件处置。

整体体验应体现“轻量、清晰、可信、智能”的后台产品气质。界面参考图 3 的浅色仪表盘风格，以浅蓝外背景、白色主工作区、柔和卡片、彩色状态标识和清晰数据层级为主要视觉特征。

## 2. 品牌与 Logo

### 推荐品牌名

**智慧交通拥堵预警**

推荐英文辅助名：

**RoadSense AI**

命名理由：

- “智”体现 AI 分析、智能客服、拥堵识别等智能化能力。
- “路”明确交通路网场景。
- “预警”覆盖实时监控、异常记录、拥堵识别和管理员推送。
- 中文名短，适合放在侧栏和顶部栏，也适合毕业设计展示。

### Logo 概念

不再使用单字母 `T` 作为图标，改为“道路 + 监控点 + 预警信号”的组合标志。

Logo 图形建议：

- 主体为两条弯曲道路线，形成向前延伸的路网透视感。
- 中心放置一个圆形监控点，表示摄像头、路口或检测节点。
- 右上角加入小型信号波纹或预警闪光，表示异常识别和消息提醒。
- 整体轮廓保持圆润，风格轻量，不做厚重盾牌或传统政务徽章。

### Logo 色彩

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 主色 | `#8BC8F7` | 浅天空蓝，呼应图 3 背景与导航高亮 |
| 辅色 | `#E59BE8` | 淡紫粉，用于 AI、智能分析、品牌记忆点 |
| 强调色 | `#71E0B4` | 绿色，用于在线、正常、通行状态 |
| 文字色 | `#1D2433` | 品牌文字和主要内容 |

### Logo 使用规范

- 左上角显示：Logo 图形 + “智路预警”。
- 侧栏收起时只显示 Logo 图形，不显示文字。
- 顶部栏或登录页可使用完整组合标识。
- 禁止仅用 `T`、普通摄像头图标或交通灯图标代替品牌 Logo。

## 5. 布局规范

### 应用框架

整体布局建议：

- 最外层：浅蓝背景，四周保留 20px-32px 空白。
- 内层：白色圆角应用容器，占据主要视口。
- 左侧：白色侧栏，宽度 240px。
- 右侧：顶部栏 + 内容区。
- 内容区：浅灰白背景或白色背景，卡片式组织。

### 侧栏

侧栏视觉参考图 3：

- 背景为白色，不再使用深蓝渐变。
- 当前模块使用浅蓝底色高亮。
- 导航项图标使用线性图标，文字颜色为灰蓝。
- 当前项文字为白色或深蓝，根据背景对比决定。
- 底部可显示系统状态、当前计划、服务连接状态。

侧栏模块文案：

- 系统控制台
- 用户管理
- 设备信息管理
- 实时监控
- 交通异常记录
- 拥堵识别
- AI 智能客服

### 顶部栏

顶部栏包含：

- 全局搜索框
- 报表导出入口
- 通知按钮
- 后端接口状态
- 当前用户头像和用户名

顶部栏风格：

- 高度建议 72px。
- 搜索框为浅灰底圆角输入。
- 通知按钮使用圆形浅色按钮。
- 用户头像使用真实头像或渐变圆形，不再使用硬编码字母风格。

## 6. Dashboard 设计

Dashboard 是系统总览页，应成为整个项目最重要的展示页面。

### 核心内容

建议包含以下区域：

| 区域 | 内容 |
| --- | --- |
| 欢迎面板 | 当前用户、日期、系统运行摘要、插画或路网视觉图 |
| 快捷入口 | 用户管理、设备管理、实时监控、异常记录、拥堵识别、AI 客服 |
| 实时告警面板 | 当前正在发生的异常和拥堵情况 |
| 设备健康总览 | 在线设备、离线设备、故障设备、设备在线率 |
| 交通趋势图 | 每周/月/年交通流量折线图 |
| 日历面板 | 点击日期查看当天最拥堵、异常最多路段 |
| 设备在线占比 | 环形图展示在线/离线比例 |
| AI 分析占比 | AI 已分析事件数、未分析事件数 |
| 统计卡片 | 用户数量、设备数、事件记录数、待处理数 |
| 地图热力图 | 展示全市或示例区域交通拥堵分布 |

### Dashboard 指标建议

| 指标 | 示例 |
| --- | --- |
| 今日异常事件 | `12` |
| 当前拥堵路段 | `5` |
| 设备在线率 | `86%` |
| AI 分析覆盖率 | `72%` |
| 待处理预警 | `4` |
| 用户数量 | `2` |
| 设备总数 | `8` |
| 交通事件记录数 | `128` |

### 卡片风格

- 普通统计卡：白底、柔和阴影、左上角图标圆点。
- 高风险卡：淡红背景，不使用刺眼纯红。
- AI 卡片：淡紫粉或深色卡片，用于突出智能分析。
- 设备健康卡：环形图 + 简短状态列表。

## 10. 交通异常记录

该模块建议升级为“异常事件处理中心”。

### 功能范围

- 汇总实时监控产生的异常信息。
- 管理员查看异常详情。
- 管理员处理事件。
- AI 展示事件原因分析和处置建议。
- 支持模拟短信/站内信推送给管理员。

### 优先级标记

| 等级 | 颜色 | 示例 |
| --- | --- | --- |
| 紧急 | 红色 | 严重拥堵、事故、道路阻断 |
| 重要 | 黄色 | 持续缓行、设备异常、疑似违停 |
| 一般 | 蓝色/灰色 | 短时流量升高、普通提醒 |

### 推送设置弹窗

参考图 1，异常事件记录模块可以增加“通知设置”弹窗：

- 是否启用短信通知。
- 冷却时间，避免重复发送。
- 通知接收人，支持管理员多选。
- 短信发送记录。
- 空状态展示“暂无历史记录”。
- 底部操作：取消、确认。

### 事件处理状态

| 状态 | 说明 |
| --- | --- |
| 未处理 | 新产生事件，等待管理员查看 |
| 处理中 | 管理员已接手 |
| 已处理 | 已完成处置 |
| 已忽略 | 确认为误报或无需处理 |

## 11. 拥堵识别

该模块是毕设创新点，应单独做成重点页面。

### 功能目标

- 获取实时路况信息。
- 调用深度学习模型识别拥堵状态。
- 展示路网图或地图热力图。
- 展示路段拥堵指数和拥堵等级。
- 给出拥堵原因推测和趋势预测。

### 页面布局

左侧：

- 地图或简化路网图。
- 路段按拥堵程度着色。
- 支持点击路段查看详情。

右侧：

- 当前拥堵排行。
- 模型识别结果。
- 拥堵原因分析。
- 未来短时趋势预测。

底部：

- 拥堵指数趋势图。
- 历史对比。

### 拥堵等级

| 等级 | 颜色 | 含义 |
| --- | --- | --- |
| 畅通 | 绿色 | 车速正常 |
| 基本畅通 | 浅绿 | 轻微车流 |
| 轻度拥堵 | 黄色 | 通行变慢 |
| 中度拥堵 | 橙色 | 排队明显 |
| 严重拥堵 | 红色 | 长时间排队或阻塞 |

## 12. AI 智能客服

### 功能范围

- 管理员自然语言提问。
- 查询交通状态、设备状态、异常事件。
- 通过 DeepSeek API 生成回答。
- 支持快捷问题。
- 支持上下文记忆。

### 设计建议

- 左侧为聊天列表或快捷问题。
- 右侧为对话区。
- AI 回复中可插入数据卡片、事件引用和跳转按钮。
- 重要回答标注“由 AI 生成，请结合实际情况确认”。

### 常见快捷问题

- 今天最拥堵的路段是哪几个？
- 当前有哪些未处理异常？
- 哪些设备处于离线状态？
- 最近一周交通流量趋势如何？
- 请生成今日交通异常简报。

## 15. 文案规范

### 语气

- 简洁、明确、偏管理系统。
- 避免花哨宣传语。
- AI 模块可稍微亲和，但不能像娱乐聊天产品。

### 示例

| 场景 | 推荐文案 |
| --- | --- |
| 空状态 | 暂无异常记录 |
| 加载中 | 正在加载数据... |
| 删除确认 | 确定删除该记录吗？删除后不可恢复。 |
| AI 提示 | AI 分析结果仅供参考，请结合现场情况确认。 |
| 推送成功 | 已向管理员发送异常提醒 |

## 16. 响应式规范

桌面端：

- 保持侧栏 + 顶栏 + 内容区三段式布局。
- Dashboard 使用 12 栅格或 CSS Grid。
- 右侧辅助栏宽度建议 320px-380px。

平板端：

- 侧栏可收起。
- Dashboard 右侧面板下移。
- 卡片变为两列。

移动端：

- 侧栏变为顶部横向导航或抽屉。
- 卡片单列展示。
- 表格改为横向滚动或卡片列表。

## 17. 后续界面改造优先级

建议分三步推进：

1. **统一品牌和布局**
   - 替换 `T` Logo。
   - 侧栏改为浅色风格。
   - 顶部栏统一搜索、通知、用户信息。
   - 全局颜色 Token 更新。

2. **重构 Dashboard**
   - 增加实时告警面板。

3. **补齐业务增强**
   - 新增拥堵识别模块占位页。
